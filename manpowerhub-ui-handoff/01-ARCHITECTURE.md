# 01 — Architecture

## Stack decision

| Concern | Choice | Why |
|---|---|---|
| Language | **TypeScript** (strict) | Typed props are the contract for consuming apps |
| UI base | **shadcn/ui** (Radix primitives) | Accessible, unstyled primitives we skin with our tokens |
| Styling | **Tailwind CSS** + CSS variables | Tokens live as CSS vars; Tailwind maps them to utilities |
| Bundler | **Vite (library mode)** + **tsup** for `.d.ts` | Fast, dual ESM/CJS, tree-shakeable |
| Catalog | **Storybook 8** (Vite builder) | Living docs; every state visible |
| Test | **Vitest** + Testing Library + axe | Unit + a11y |
| Icons | **lucide-react** | The reference icon set (`reference/src/icons.jsx`) is lucide-style, stroke 1.6 |
| Fonts | **Geist** (display/body) + **JetBrains Mono** | Already used in the reference |
| Package mgr | **pnpm** workspaces | Monorepo, strict deps |

> shadcn/ui is **copied in**, not a dependency. We own the component code and
> restyle it with tokens. Keep the shadcn file structure (`components/ui/*`) but
> our public API re-exports from `packages/ui/src/index.ts`.

## Package boundaries

- **`@manpowerhub/tokens`** has zero React. It exports `globals.css`, the Tailwind
  `preset`, and typed token objects. Swapping this package (or its CSS vars)
  reskins every app — this is the multi-app mechanism (see `08`).
- **`@manpowerhub/ui`** depends on `@manpowerhub/tokens` and `react`/`react-dom`
  (peer deps). Ships ESM + CJS + `.d.ts`. Side-effect-free for tree-shaking
  (`"sideEffects": ["*.css"]`).
- **Apps** install both, import `@manpowerhub/tokens/globals.css` once at the
  root, extend the Tailwind preset, wrap in `<ThemeProvider>`, then use components.

## Public API conventions

```ts
// consuming app
import { Button, Card, DataTable, AppShell, KPICard } from "@manpowerhub/ui";
import "@manpowerhub/tokens/globals.css";
```

- **Named exports only** (no default exports) — better for tree-shaking + refactors.
- Every component exports its **props type**: `export interface ButtonProps …`.
- Compound components use dot notation: `Card.Header`, `Card.Body`, `Card.Footer`; `Tabs`, `Tabs.List`, `Tabs.Trigger`, `Tabs.Content`.
- Forward refs on every leaf component (`React.forwardRef`).
- Variants via **`cva`** (class-variance-authority) — the shadcn convention.
- `className` always accepted and merged last via `cn()` so consumers can override.
- Polymorphism: support `asChild` (Radix Slot) on interactive elements.

## Naming

- Components: `PascalCase`. Files: `kebab-case.tsx`. Folders: `kebab-case`.
- Props: `camelCase`. Boolean props read as adjectives (`loading`, `disabled`, `destructive`).
- Variant prop is always `variant`; size is always `size`.
- CSS custom properties keep the reference names (`--accent`, `--fg`, …) mapped to shadcn semantic names (see `02`).

## Accessibility baseline (enforced)

- All interactive elements keyboard reachable; visible focus via `--ring` (3px ring, `--accent-ring`).
- Radix primitives provide roles/aria for Dialog, DropdownMenu, Tabs, Tooltip, Select, Checkbox, Switch — don't strip them.
- Color is never the only signal (StatusBadge pairs color with a label + dot).
- Icons that convey meaning get `aria-label`; decorative icons get `aria-hidden`.
- Respect `prefers-reduced-motion` — gate non-essential motion.
- Storybook `@storybook/addon-a11y` must pass on every story.

## Testing

- **Unit** (Vitest + Testing Library): render, variant classes, disabled/loading behavior, callbacks fire.
- **a11y** (`vitest-axe` or Storybook test runner): no violations on default + key states.
- **Interaction** (Storybook `play` functions): ⌘K opens/filters/navigates; Dialog focus trap; Tabs keyboard.
- **Visual** (optional): Chromatic on the Storybook build for light + dark.

## Versioning & release

- **Changesets** for semver + changelogs. `@manpowerhub/tokens` and `@manpowerhub/ui` versioned independently.
- Publish to a private registry (GitHub Packages / npm org). Prerelease with `next` tag.
- Breaking token changes = major on `tokens`; document migration in the changeset.

## Binding rules — paste into repo `CLAUDE.md`

```md
# ManpowerHub UI — binding rules

- Build on @manpowerhub/ui + @manpowerhub/tokens. Don't hand-roll components that exist.
- Styling comes from tokens (Tailwind semantic classes: bg-background, text-foreground,
  border-border, bg-primary, text-muted-foreground, ...). Never hardcode hex colors.
- shadcn/ui primitives live in packages/ui/src/ui; our styled components wrap them.
- Every component: TS props exported, forwardRef, cva variants, className merged via cn().
- Every async surface: loading (skeleton/spinner), empty, and error states.
- Works in light AND dark (.dark on <html>). Test both.
- Keyboard + screen-reader accessible; visible focus rings. axe must pass.
- New/changed components need a *.stories.tsx covering all variants + states.
- Full spec: docs/ui-handoff/. Reference prototype: docs/ui-handoff/reference/.
```
