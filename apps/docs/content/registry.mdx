# Install via registry

Components and blocks are also distributed as source you copy into your own
repo via the standard [shadcn CLI](https://ui.shadcn.com) — there is no
`@manpowerhub/*` npm package to install for this path. The registry serves
component and block definitions from the docs site on Vercel (`out-nu-vert.vercel.app`).

## One component

```bash
npx shadcn@latest add https://out-nu-vert.vercel.app/r/button.json
```

## A full block (pulls its component deps automatically)

```bash
npx shadcn@latest add https://out-nu-vert.vercel.app/r/pricing-table.json
```

This copies `pricing-table.tsx` plus its dependencies — `card`, `button`,
`badge`, and the `cn` util — into your project, and installs the required npm
packages (`class-variance-authority`, `lucide-react`, the relevant
`@radix-ui/*` packages). Registry dependencies are resolved as full registry
URLs, so nested component deps come along automatically — verified end-to-end
against a fresh Next.js + Tailwind app, where the block lands and typechecks
with no manual follow-up.

## One-time setup: tokens

Components use CSS variables and a Tailwind preset, so there's a one-time
setup step per consumer app:

- Install the tokens `globals.css` and import it once in your root layout.
- Add the tokens Tailwind preset to your `tailwind.config` `presets: [...]`.

React and Tailwind are peer prerequisites your app already owns — the
registry does not install or configure either.
