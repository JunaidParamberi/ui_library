## Usage

Pass controlled `email` and `password` field objects and an `onSubmit` handler; the form
does not manage submission state itself. Set `loading` while a request is in flight to
disable the submit button and show a spinner, and pass `error` to surface a server-side
validation or auth failure above the fields. Use `mode="signup"` to switch the heading and
submit label.

## Accessibility

The error message is rendered with `role="alert"` so screen readers announce it as soon as
it appears. Fields are associated with their labels via generated `id`s, and the submit
button exposes `aria-busy` while `loading`. All controls are native form elements and are
keyboard-operable without additional wiring.
