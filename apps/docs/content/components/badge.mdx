## Usage

Use semantic variants (`success`, `warning`, `danger`, `info`) to communicate status, and
`accent` or `outline` for neutral labels. Add `dot` for a compact status indicator when space
is tight. Badges are non-interactive — don't put actions inside them.

## Accessibility

Renders a `<span>`, so it carries no implicit role or focus behavior. Badges are decorative
labels, not controls — if a badge communicates state that matters to screen reader users (e.g.
"Overdue"), make sure the surrounding text also conveys it, since color alone isn't
announced.
