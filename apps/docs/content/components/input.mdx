## Usage

Wrap `Input` in `Field` to attach a `label`, `hint`, `error`, and `required` marker. Set
`invalid` (or pass `error` to `Field`) when validation fails — it switches the border and
focus ring to the destructive color and sets `aria-invalid`. Use `inputSize="lg"` for
prominent, standalone forms.

## Accessibility

Renders a native `<input>`, so it's keyboard-focusable and works with browser autofill. Always
pair it with a `Field`-provided `<label htmlFor>` rather than a placeholder alone. When
`invalid`, `aria-invalid="true"` is set automatically. `Field`'s `error` text renders with
`role="alert"` so it's announced when it appears; pass `invalid` to `Input` alongside `error`
on `Field` to keep the visual and semantic states in sync.
