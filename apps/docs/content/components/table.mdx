## Usage

For static markup, compose the primitives directly (`Table` > `TableHeader`/`TableBody` >
`TableRow` > `TableHead`/`TableCell`). For sortable, dynamic data, use `DataTable<T>` with a
`columns` array (each with `key`, `header`, optional `render`, and `sortable`), `data`, and a
`getRowId` function. Clicking a sortable header cycles ascending → descending → unsorted.

## Accessibility

Renders a native `<table>` with proper `<thead>`/`<tbody>`/`<tr>` structure, so row and column
relationships are conveyed to assistive tech without extra ARIA. `DataTable`'s sortable
headers render as buttons — activate with Enter/Space — and the header cell exposes the
current direction via `aria-sort` (`"ascending"`/`"descending"`/`"none"`), so assistive tech
announces sort state changes.
