## Usage

Compose `Dialog` > `DialogTrigger` (`asChild` around your trigger element) > `DialogContent`,
using `DialogHeader`/`DialogTitle`/`DialogDescription` for the heading and
`DialogFooter`/`DialogClose` for actions. Reserve dialogs for tasks that need the user's full
attention (confirmations, short forms); for lightweight menus prefer `DropdownMenu`, and for
non-blocking info prefer `Tooltip` or inline text.

## Accessibility

Built on Radix `Dialog`: content renders with `role="dialog"` and `aria-modal="true"`, focus is
trapped inside while open and restored to the trigger on close, background content is hidden
from assistive tech, and Escape (or the built-in close button) dismisses it. Always include a
`DialogTitle` — use the `sr-only` utility if it shouldn't be visually shown — since it's what
screen readers announce when the dialog opens.
