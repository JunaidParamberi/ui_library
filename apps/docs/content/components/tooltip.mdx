## Usage

Wrap the app (or the section using tooltips) in a single `TooltipProvider`. Compose
`Tooltip` > `TooltipTrigger` (usually `asChild` around an icon button) > `TooltipContent`.
Keep tooltip text short — a label or one clause — and never put essential information only in
a tooltip, since it isn't available on touch devices without a long-press.

## Accessibility

Built on Radix `Tooltip`: content is shown on both hover and keyboard focus of the trigger, and
dismissed with Escape, so keyboard users get the same information as mouse users. The trigger
is linked to the tooltip content via `aria-describedby`, and the tooltip is not itself
focusable or interactive.
