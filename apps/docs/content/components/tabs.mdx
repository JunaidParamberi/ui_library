## Usage

Compose `Tabs` (set `defaultValue` or control with `value`/`onValueChange`), `TabsList`,
`TabsTrigger` (one per panel, matching `value`), and `TabsContent`. `TabsTrigger` accepts an
optional `count` badge for showing item totals. Use tabs for switching between peer views of
the same object, not for multi-step flows — use a stepper for those.

## Accessibility

Built on Radix `Tabs`, implementing the ARIA tabs pattern: `TabsList` has `role="tablist"`,
triggers use `role="tab"` with roving tabindex, and arrow keys move focus between tabs while
Enter/Space activates the focused one. The active panel is exposed via `aria-selected` and
matched to its `TabsContent` by `aria-controls`/`id`.
