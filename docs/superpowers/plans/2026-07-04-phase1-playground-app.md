# Playground App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `apps/playground` — a deployable Vite + React sandbox that consumes `@manpowerhub/ui` and `@manpowerhub/tokens` via workspace deps, renders themed components, and runs on `pnpm dev`.

**Architecture:** New workspace package under `apps/*` (already globbed by `pnpm-workspace.yaml`). Vite library-consumer app. Tailwind extends the tokens preset; PostCSS uses `postcss-import` so `@manpowerhub/tokens/globals.css` is inlined before Tailwind runs (same wiring as `packages/ui`). Component utility classes are generated by scanning `packages/ui/src` (and future `packages/blocks/src`) via Tailwind `content` globs. A Vitest smoke test asserts the app renders a themed component.

**Tech Stack:** Vite 5, `@vitejs/plugin-react`, React 18, Tailwind CSS 3.4, `postcss-import`, Vitest + Testing Library + jsdom, TypeScript strict.

## Global Constraints

- Package manager: **pnpm** workspaces; internal deps use `workspace:*`.
- Workspace globs already include `apps/*` (`pnpm-workspace.yaml`) — no change needed there.
- React is a **peer** of `@manpowerhub/ui` (`^18.3.0`); playground installs `react@^18.3.1` + `react-dom@^18.3.1` directly.
- Tokens preset has peer deps `tailwindcss@^3.4.0` + `tailwindcss-animate@^1.0.7`; playground installs both.
- TypeScript: strict, `moduleResolution: "Bundler"`, `jsx: "react-jsx"` — inherited from `tsconfig.base.json`.
- No hardcoded hex colors in playground pages — semantic Tailwind classes / component props only (mirrors CLAUDE.md rule; keeps playground a faithful consumer).
- Playground package is `private: true` — never published.

- [ ] **Step 6: Create `apps/playground/src/App.tsx`**

```tsx
import { useState } from "react";
import { KitchenSink } from "./pages/kitchen-sink";

type Page = { id: string; label: string; element: React.ReactNode };

const PAGES: Page[] = [
  { id: "kitchen-sink", label: "Kitchen Sink", element: <KitchenSink /> },
];

export function App() {
  const [active, setActive] = useState(PAGES[0]!.id);
  const current = PAGES.find((p) => p.id === active) ?? PAGES[0]!;

  return (
    <div className="flex min-h-screen bg-background text-foreground">
      <nav className="w-56 shrink-0 border-r border-border p-4 space-y-1">
        <div className="mb-4 font-display text-lg">Playground</div>
        {PAGES.map((p) => (
          <button
            key={p.id}
            onClick={() => setActive(p.id)}
            className="block w-full rounded-md px-3 py-2 text-left text-sm text-muted-foreground hover:bg-secondary aria-[current=true]:bg-secondary aria-[current=true]:text-foreground"
            aria-current={p.id === active}
          >
            {p.label}
          </button>
        ))}
      </nav>
      <main className="flex-1 p-8">{current.element}</main>
    </div>
  );
}
```
