# Docs App (`apps/docs`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `apps/docs` — a Nextra documentation site on Next.js 15 + Nextra 4 that consumes workspace packages, renders live demos, links to Storybook, supports light/dark via token CSS vars, devs on port 3001, and static-exports cleanly.

**Architecture:** Nextra 4 content-directory mode with `app/[[...mdxPath]]/page.tsx`. Theming via `globals.css` inlining `@manpowerhub/tokens/globals.css` + `nextra-theme-docs/style.css` before `@tailwind` directives (postcss-import). Interactive demos are `"use client"` components loaded via `dynamic(..., { ssr: false })` in `src/components/dynamic-demos.tsx`. Nextra `Layout` is wrapped in a client boundary (`NextraDocsLayout`) with a pnpm patch for `nextra-theme-docs@4.6.1` Layout zod validation.

**Tech Stack:** Next.js 15, Nextra 4.6, `nextra-theme-docs`, React 18, Tailwind CSS 3.4, postcss-import, Pagefind.

## Global Constraints

- Package name: `@manpowerhub/docs`, `private: true`, dev port **3001**.
- Static export: `output: "export"`, `images.unoptimized: true`.
- Never import `@manpowerhub/tokens` in `.tsx` — CSS/Tailwind only.
- Do not modify `packages/ui`, `packages/blocks`, or `packages/tokens`.
- Storybook: UI `:6006`, blocks `:6007`.

## Implementation notes (discovered during build)

1. **Next.js 15** required (Nextra 4.6 + App Router).
2. **PostCSS:** use `postcss.config.mjs` object format; resolve token + nextra CSS paths in `postcss-import`.
3. **globals.css order:** `@import tokens` → `@import nextra-theme-docs/style.css` → `@tailwind` directives.
4. **pnpm patch** `nextra-theme-docs@4.6.1`: `LayoutPropsSchema.safeParse({ ...themeConfig, children })`.
5. **MDX Wrapper:** omit `getMDXComponents().wrapper` during SSG — causes `Missing ConfigContext.Provider` on pages with client demos.
6. **Demos:** import from `dynamic-demos.tsx` (ssr: false), not raw demo files in MDX.
